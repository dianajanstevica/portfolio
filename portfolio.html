#1 Basic web

Responsive website

My role: UI design

Link to the full project >

Project description: First responsive website I've ever done, a very simple one, dedicated to Postmodernism style.

In this theme I learned how to use basic html and css, as well as basic design principles. I've also learned how to make webpages responsive for different devices using css media queries and how to use semantically correct code.

I definitely didn't consider this as an easy task at the moment of making the project as my UI skills at that moment were close to non-existent. Still I managed to code my first website in a quite short amount of time but of course it required a lot of hard work and research. Looking back at it, it feels like after the project I had a solid base for basic html and css, which has helped me a lot in further projects.

#3 basic content

A bicycle guide for non-danes

My role: Video production

Link to team canvas >

Link to storyboard >

Link to Xd prototype >

Link to final production >

Project description: A study case for the Municipality of Copenhagen - a video guide for foreigners about biking in Copenhagen + microsite with relevant information.

In this project I was responsible for capturing the video, as well as editing both video and sound. In this theme I used the principles introduced to us such as composition, diffent types of shots ( close-ups, medium ang long shots ), camera movements ( static, panning, dolly ), 180 degree rule, handheld/stabilsed shots etc. I learned how to use Adobe Premiere for video editing and colour correction, Adobe Audition for sound work and setting up camera and sound gear.

Personally, this has been my favourite project so far, I have experience with photography, but video production has a whole other way of telling a story as it includes a lot more aspects to it. I had a great team that helped me in dealing with time management as they set very strict deadlines and we had a great respect towards each other which resulted in a good work flow. This project improved my presentation skills as well because it was pitched in front of the whole class.

#4 Basic ux

design sprint

My role: UX design and prototyping

Link to Design Sprint documentation >

Link to Xd prototype >

Project description: Study case for Frederikke Bender, Head of Multimedia Design and Communication at KEA about developing a user friendly solution for KEA's intranet Fronter.

The project was a 5 day sprint with an end goal of a tested Xd prototype. In this project I contributed with researching the existing solution as well as developing a user friendly web app prototype for Fronter - KEA's existing intranet.

This project allowed me to extend my knowledge in prototyping in Adobe Xd, document a research for existing solutions, follow KEA's design guide as well as run various user tests all in a very short time span. I consider design sprints a very useful tool for recieving a quick test ready prototypes as it requires a lot of quick decision making instead of endless discussions and planning. It's also very interesting to see how people produce an idea in only 5 days.
